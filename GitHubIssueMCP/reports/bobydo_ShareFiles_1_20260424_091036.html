Project Requirements Build an application that integrates with a local LLM server (no external AI services). Retrieve issue data (URL + content) using either: - a specific issue number, or - issues created/updated today. Analyze each issue to determine clarity of requirements, including: - a clarity score - actionable improvement recommendations Analyze replies and classify them into: - Discussion - Proposed Solution For replies classified as Proposed Solution: - estimate technical feasibility (e.g., complexity, effort, risk) - ignore Discussion replies Store all results in a readable format (e.g., HTML report or structured output like JSON for further use)
